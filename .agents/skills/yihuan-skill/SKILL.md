# 异环资源规划

使用这个技能，根据玩家当前库存和版本目标计算异环资源方案。

## 参考文件

- 需要稳定规则时，读取 `references/fixed-rules.md`：输入与档案优先级、资源换算、抽卡成本、礼包优先级、好感度体系、活动支付原则。
- 需要分析 1.1 版本时，读取 `references/version-1.1.md`：1.1 收入、联动活动档位、常见开销和组合分析方法。
- 需要玩家长期偏好时，读取 `profiles/<档案名>.md`。如果用户没有指定档案，默认使用 `profiles/nineo.md`。
- 默认生成 HTML 报告时，使用 `templates/resource-report.html`：固定版式、样式、模块顺序和术语。

## 工作流程

1. 确认版本。用户没有说明版本时，询问版本；如果请求明显指向唯一可用版本，可直接使用该版本参考。
2. 按 `fixed-rules.md` 的输入与档案优先级确认玩家参数；档案缺失时继续使用用户输入，并在输出假设中说明没有读取到档案。
3. 判断资源口径：当用户说“版本 X + 当前资源”时，默认理解为上一版本末库存，用来规划版本 X，需要叠加版本 X 收入；只有用户明确说“当前剩余资源”“已领取后的余量”“版本进行中余量”时，才不重复叠加整期版本收入。
4. 读取 `fixed-rules.md`。
5. 读取对应版本参考文件。
6. 统一术语：
   - 使用 `限定骰子` 表示限定角色池抽卡道具。
   - 使用 `常驻骰子` 表示常驻池抽卡道具。
   - 除非用户明确说明可用，否则不要把 `常驻骰子` 计入限定角色或武器目标。
7. 根据档案或用户输入选择版本收入档位：零氪、小月卡、大小月卡或用户提供的自定义收入。
8. 计算版本可用资源：当前库存 + 版本收入 + 已选择礼包收入。
9. 计算目标消耗：限定角色抽卡、武器抽卡、活动支付、好感度消耗、方斯消费。好感度消耗按 `每日消耗 x 版本天数` 计入版本总消耗。
   - 目标消耗项目名统一写作：`角色 X 个`、`武器 X 把`、`猎人商店`、`好感度`、`保时捷联动`、`买房`。
10. 计算版本活动支付方案：
   - 活动档位、支付规则和组合方法以对应版本参考文件为准。
   - 如果活动可混合支付，按版本文件提供的公式或档位表计算若干可执行组合。
   - 根据用户底仓要求、环石余量、方斯余量，给出推荐方案和替代方案。
   - 不把某个版本的特殊活动规则写入通用工作流程。
11. 生成执行顺序建议：
   - 执行顺序是建议，不是固定锁死流程；可按用户偏好、卡池开放时间、活动截止时间和实际库存调整。
   - 默认优先级：限定角色抽卡 > 武器目标 > 保时捷联动 > 好感度/买房 > 礼包与活力补差。
   - 好感度和买房视为同级目标；两者都低于保时捷联动。若资源紧张，先保保时捷联动，再在好感度与买房之间按用户偏好调顺序。
12. 默认直接生成 HTML 报告：复制 `templates/resource-report.html` 的结构和样式，只替换占位内容与表格行；保留模块顺序、配色、卡片和表格样式。HTML 报告中统一使用 `版本可用资源`、`版本末环石`、`版本末方斯`，不要使用 `总可用资源` 或 `期末`。
   - HTML 的 `<title>` 和 `<h1>` 统一写作：`档案名 v版本 资源分析`，例如 `yang v1.1 资源分析`。
   - HTML 文件默认生成到项目根目录的 `reports/` 路径下。
   - HTML 文件名统一写作：`name-date.html`，例如 `yang-20260528.html`；`name` 使用档案名或用户指定名称，`date` 使用生成日期的 `yyyyMMdd`。
13. 只有用户明确要求纯文本、Markdown 或不生成文件时，才输出精简文本方案：结论、版本可用资源、消耗资源、联动方案分析、执行方案、风险。

## 默认 HTML 报告格式

除非用户要求其他格式，否则生成 HTML 文件，并按 `templates/resource-report.html` 的结构输出：

- 标题：`档案名 v版本 资源分析`，同时用于 `<title>` 和 `<h1>`。
- 文件：默认保存到 `reports/name-date.html`，日期格式为 `yyyyMMdd`。
- 结论：是否能完成目标、推荐联动从哪一档开始用环石、版本末核心余量。
- 版本可用资源：当前库存、版本收入、合计。
- 消耗资源：按统一项目名列出 `角色 X 个`、`武器 X 把`、`猎人商店`、`好感度`、`保时捷联动`、`买房` 等目标消耗、方斯开销和备注。
- 联动方案分析：只列出连续后段方案，从某一档开始，到最后一档全部用环石支付，给出 3-6 条可选方案和版本末余量。
- 执行顺序建议：用 3-5 条写清抽卡、联动、好感度/买房同级处理和礼包补差顺序；说明该顺序可调整。礼包建议合并在这里，不单独成段。
- 风险：只列出最关键风险，通常包括抽卡黑于期望、额外都市消费、活动规则变化。

```markdown
| 资源 | 当前 | 1.1 收入 | 合计 |
| --- | ---: | ---: | ---: |

**消耗资源**
| 项目 | 消耗 | 备注 |
| --- | ---: | --- |

**联动方案分析**
只列出连续后段方案：从某一档开始，到最后一档全部用环石支付。给出 3-6 条可选方案和版本末余量。

**执行方案**
用 3-5 条写清抽卡、联动、好感度/买房同级处理和礼包补差顺序，并说明该顺序可调整。礼包建议合并在这里，不单独成段。

**风险**
只列出最关键风险：抽卡黑于期望、额外都市消费、活动规则变化。
```

## 用户输入模板

如果用户需要一次新的分析，询问或自行推断：

```text
档案：默认 nineo / 指定档案名
版本：
当前库存：方斯 / 环石 / 限定骰子 / 常驻骰子 / 三重钥匙 / 其他票
付费档位：零氪 / 小月卡 / 大小月卡 / 礼包预算 / 双倍是否可用
目标：角色几个 / 武器几把 / 是否参加活动 / 都市消费
特殊活动：活动档位、是否可混合支付、限购礼包
底仓要求：希望版本结束保留多少方斯和环石
```

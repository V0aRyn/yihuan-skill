# 异环 1.1 版本主资料

本文件属于版本事实层，用于记录 1.1 版本的特殊规则、活动档位、常见目标和计算公式，不设置固定玩家偏好或固定当前库存。

版本日期、资源收入、日期拆分、礼包上架和汇总校验见 `progress.md`；分析时优先读取 `progress.md` 的资源数据。

支付原则：

- 档位 1-15 必须按顺序购买；每个已购买档位可单独选择环石或方斯支付。
- 默认只把 `1 万方斯等效环石 < 8` 的档位纳入环石替换分析，并按等效值从低到高逐步加入组合。
- `1 万方斯等效环石 >= 8` 的档位默认保留方斯支付；只有用户指定强保方斯、环石充足或方斯底线仍不足时，才列扩展补差方案。
- 若低成本方斯礼包能解决缺口，先列礼包补差，再考虑双倍首充。

### 默认组合分析

默认组合顺序：

```text
候选档位 = 1 万方斯等效环石 < 8 的档位
组合顺序 = 按 1 万方斯等效环石从低到高排列：15 -> 14 -> 13 -> 12 -> 11 -> 8
联动环石 = 环石支付档位的环石合计
联动方斯 = 其余已购买档位的方斯合计
```

| 组合 | 环石支付档位 | 联动环石 | 联动方斯 | 适合情况 |
| --- | --- | ---: | ---: | --- |
| 全方斯基线 | 无 | 0 | 19,880,000 | 环石极缺、方斯极富裕 |
| 5.238 起 | 15 | 2,200 | 15,680,000 | 优先使用最划算档位保方斯 |
| 5.714 起 | 15、14 | 4,200 | 12,180,000 | 方斯压力中等 |
| 6 起 | 15、14、13 | 6,000 | 9,180,000 | 需要明显保方斯 |
| 6.4 起 | 15、14、13、12 | 7,600 | 6,680,000 | 方斯压力较大 |
| 7 起 | 15、14、13、12、11 | 9,000 | 4,680,000 | 强保方斯 |
| 7.5 起 | 15、14、13、12、11、8 | 9,600 | 3,880,000 | 接近 `<8` 档位全环石 |

输出时选 3-6 条与约束最相关的方案；若用户指定环石上限或方斯底线，围绕约束列相邻方案。若 `<8` 档位全改环石后仍不满足方斯底线，再补列 `1 万方斯等效环石 = 8` 的 9、10 档或全环石方案，并标记为扩展补差。报告模块标题写作 `保时捷联动方案分析`。

## 1.1 目标消耗事实

### 常见开销

以下开销只在用户目标、档案策略或固定规则要求时计入：

| 项目 | 消耗 | 计入条件 |
| --- | ---: | --- |
| 买房 | 7,500,000 | 用户目标包含买房时计入 |
| 猎人交易所 | 2,537,000 | 常规购买；用户明确不买时排除 |
| 好感道具日常 | 51,000/天 x 35 天 = 1,785,000 | 用户目标或档案策略要求日常好感投入时计入 |
| 三项全计入合计 | 11,822,000 | 买房、猎人交易所和好感道具日常均计入时使用 |

### 目标模板

按用户输入选择目标；用户本次输入优先于档案策略。

| 目标 | 计算规则 | 备注 |
| --- | --- | --- |
| 角色 X 个 | 按用户指定数量计算；默认 45 抽/角色 | “角色全拿”按 `progress.md` 限定棋盘时间顺序匹配 |
| 武器 X 把 | 按用户指定把数计算；默认 60 抽/把 | 指定武器名时匹配 `progress.md` 同名弧盘研摹 |
| 猎人交易所 | 2,537,000 方斯 | 获得 500 环石 + 40 三重钥匙 + 5 永恒的螺音 |
| 保时捷联动 | 按本文件“默认组合分析”列方案 | 可规划满档或用户指定档位 |
| 好感度 | 按用户输入或档案策略计入 | 备注写公式，如 `15000 x 2 + 3000 x 7 = 51000/天` |
| 买房 | 7,500,000 方斯 | 仅用户目标包含买房时计入 |
| 礼包 | 根据用户本次氪金预算判断是否列入 | 已确认购买才计入资源；备选礼包只列说明 |
| 双倍首充 | 根据用户本次预算判断是否列入 | 未明确购买时不计入 |

### 抽卡期望与补差

抽卡期望：

```text
1 个角色：45 抽
1 把武器：60 抽
```

补差公式：

```text
角色补抽所需环石 = max(0, 角色目标数量 x 45 - 可用限定骰子) x 160
武器补差所需环石 = max(0, 武器目标把数 x 60 - 可用三重钥匙) x 160
```
